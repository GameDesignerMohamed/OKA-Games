# Data Vault

**Concept:** Top-down stealth heist where the player infiltrates Google's server room to steal back their own behavioral data blocks and carry the growing chain to the exit alive.
**Player fantasy:** Your data is yours — reclaim it, carry it, don't let the platform win.

## Core Loop
Move through a procedural server room grid → grab labeled data blocks (each shows a real behavioral data type: "SESSION #4: replayed lvl2 × 7", "LOCATION: Cairo, 02:04") → chain trails visually behind player → reach the EXIT with 5+ blocks before guards detect you. Sprint costs you noise radius — speed trades privacy. Decoy mechanic sacrifices one block to distract a guard: a values decision inside the game.

## Key Mechanics
- Labeled data blocks — ownership signal works as mechanic, not juice
- Sprint/noise tradeoff — convenience at cost of sovereignty (maps Google Level Up thesis directly)
- Decoy drop (Space) — choose which piece of your behavioral record to sacrifice
- Dual guard detection: spotlight cone (visual) + audio radius (sprint noise)
- Seeded procedural grid — consistent per-level layouts, skill compounds

## What's Built
Working prototype proves the ownership signal. Death screen "DATA RETURNED TO GOOGLE" is the strongest loss copy in OKA catalog — platform wins when player dies. Win screen needs upgrade: current badge list deflates the arc; condition is a visual chain ledger (connected nodes, glow, spatial layout) that matches the escape climax the game builds toward.
